## 0.5. 저자 프로필 — 사이트의 최대 무기 (E-E-A-T)

구글이 정보성 사이트를 평가하는 핵심 기준이 E-E-A-T(경험·전문성·권위·신뢰)다. 저자는 이 네 가지를 실제로 갖췄고, 이것이 이 사이트가 수많은 "카더라" 주재원 블로그와 갈리는 지점이다. 소개 페이지·글 하단 저자 박스에 아래를 **사실 그대로** 녹인다(과장 금지, 회사명은 필요시 익명화 가능).

**실제 이력 (이력서 기반):**

**표기 방침 (확정): 회사명은 공개하지 않는다. 역할·국가·기간·전문 배경은 유지한다.**

| 국가 | 소속(익명화)/역할 | 기간 | 이 경험이 만드는 콘텐츠 |
|---|---|---|---|
| 대만 | 제조 대기업 재무주재원 | 2022.12–2024.06 | 대만 부임·이사(견적서 실물 보유)·은행·세무 실전 |
| 멕시코 | 제조 대기업 재무주재원 | 2024.06–2025.12 | 멕시코(중남미) 부임·치안·노무·현지 생활 |
| 폴란드 | 렌탈·물류 법인 경영관리 팀장 | 2025.12–현재 | 유럽 부임·비자·거주등록·정착(현재진행형) |

**전문성:**
- 경영학 학사 + KMBA(경영대학원) — 재무·회계 정식 교육 배경
- 총 재무경력 12년+ (K-IFRS 결산, 자금·환율 운용, 세무 대응, FP&A)
- 해외법인에서 **재무·세무·인사·총무를 단독 총괄** — 주재원이 현지에서 부딪히는 행정·돈 문제를 업으로 다뤘음
- 외화 자금(USD·EUR·AUD·TWD·MXN·PLN) 실무 → "환율·송금·환전" 콘텐츠의 진짜 근거

**이 프로필이 주는 3가지 전략적 이점:**
1. **돈·세금 카테고리가 독보적 차별화 무기가 된다.** 일반 주재원 블로거는 세금·환율을 못 쓴다. 저자는 이게 본업이다 → 3장에서 이 카테고리를 5개로 확대.
2. **"3개국 비교" 콘텐츠는 사실상 경쟁자가 없다.** 대만 vs 멕시코 vs 폴란드를 한 사람이 실제 비교하는 글은 검색에서 유일할 가능성이 높다 → 3장에 비교글 신설.
3. **애드센스 심사 통과율이 올라간다.** 심사는 "이 사이트가 신뢰할 만한 실제 저자가 만든 유용한 콘텐츠인가"를 본다. 구체적 경력·재직 이력·전문 배경이 소개 페이지에 있으면 유리하다.

> **확정:** 회사 실명은 전 페이지에서 공개하지 않는다("제조 대기업", "렌탈·물류 법인" 식 익명화). 단 **역할(재무주재원/경영관리 팀장)·국가·기간·전문 배경(재무·MBA)은 유지**한다. 익명화해도 E-E-A-T 효과는 대부분 살아있다.

## 1. 타깃 독자와 검색 시나리오

### 페르소나

| 페르소나 | 상황 | 검색하는 것 |
|---|---|---|
| A. 발령 직후 (핵심) | 3~6개월 뒤 부임 예정, 뭐부터 할지 모름 | "주재원 준비", "해외 부임 체크리스트", "국제이사 비용" |
| B. 부임 초기 | 현지 도착 1~3개월, 정착 실무 중 | "해외 은행 계좌 개설", "폴란드 비자", "해외 eSIM" |
| C. 동반 가족 | 배우자·자녀 동반, 생활 준비 담당 | "주재원 자녀 학교", "해외이사 짐 목록", "주재원 배우자" |
| D. 재무·회계 담당 (틈새·고가치) | 본인이 법인 재무를 맡거나 세금이 궁금한 주재원 | "주재원 세금", "해외 근로소득 신고", "이전가격", "법인 자금" |

> 페르소나 D는 검색량은 작지만 경쟁이 거의 없고, 저자가 압도적으로 잘 쓸 수 있는 영역이다. 애드센스 단가(CPC)도 금융·세무 키워드가 높다. 소수 정예 트래픽 + 고단가 = 방치형 사이트에 유리.

### 콘텐츠 원칙 (전부 확정)

1. **경험 없는 내용은 싣지 않는다.** 초안은 Claude 생성 → 본인 경험·수치·실수담으로 반드시 수정
2. 모든 글은 "검색 질문 하나에 답 하나" 구조 — 글 제목 = 검색어 형태
3. **3개국(대만·멕시코·폴란드)을 자산으로 쓴다.** 각 글은 "주재원 일반" 프레임으로 쓰되, 저자가 실제 겪은 국가의 구체 사례·수치를 근거로 넣는다. 국가는 태그로 관리해 추후 국가별 허브 페이지로 확장한다.
4. **비교가 곧 콘텐츠다.** "대만 vs 멕시코 vs 폴란드"처럼 한 사람이 세 나라를 실제 비교하는 글은 경쟁자가 거의 없다. 이 각도를 최대한 활용한다.
5. **돈·세금은 근거를 드러낸다.** 재무 담당으로서 직접 처리한 경험임을 글에 명시(단, 특정 회사 내부정보·수치는 배제하고 일반화). 제도·세금 글에는 면책 고지를 자동 삽입한다.
4. 각 글에 최종 수정일 표기 (정보성 글 신뢰도 + 재방문 색인)

## 2. 사이트 구조 (IA)

### 메뉴 (4카테고리 + 고정 페이지)

```
주재원노트
├── 부임 전 준비 (before)      ← 페르소나 A
├── 현지 정착 (settle)          ← 페르소나 B
├── 돈·세금 (money)             ← 본인 전문 분야, 차별화 포인트
├── 가족·생활 (family)          ← 페르소나 C
└── 고정: 소개(about) · 개인정보처리방침(privacy) · 연락처(contact)
```

### URL 규칙

- 글: `/{카테고리}/{slug}.html` — 예: `/before/moving-cost.html`
- slug는 영문 소문자-하이픈. 목록: `/before/index.html` 등 카테고리별 인덱스
- 홈(`/index.html`): 카테고리 소개 + 최신 글 + "부임 타임라인" 요약 (사이트 시그니처 콘텐츠)

## 4. 페이지 규격 (SEO 스펙, 전 페이지 공통)

| 요소 | 규격 |
|---|---|
| `<title>` | `{글 제목} — {사이트명}` (60자 이내) |
| meta description | 글별 고유 작성, 120~155자, 검색 질문에 대한 답 요약 |
| `<h1>` | 페이지당 정확히 1개 = 글 제목 |
| 본문 구조 | h2/h3 목차 구조, 글당 1,500자 이상 (한글 기준), 표·목록 적극 사용 |
| OG 태그 | og:title, og:description, og:url (SNS 공유 대비) |
| canonical | 전 페이지 자기 자신 지정 |
| 구조화 데이터 | Article JSON-LD (headline, datePublished, dateModified) |
| lang | `<html lang="ko">` |
| 글 하단 | 관련 글 3개 자동 배치 (같은 카테고리 우선 → 태그 매칭) |
| 광고 자리 | 본문 상단·중간·하단에 광고 삽입용 슬롯 주석 미리 배치 (승인 후 코드만 삽입) |


### 글 front matter 스키마

```yaml
---
title: 국제이사 견적·비용, 직접 해보고 알게 된 것
description: 국제이사 업체 견적 비교와 실제 지불 비용, 견적서에서 반드시 확인할 항목을 정리했다.
category: before
tags: [국제이사, 비용, 견적]
country: [대만]          # 저자가 실제 근거로 삼은 국가. 없으면 생략. 3개국 공통이면 [대만, 멕시코, 폴란드]
featured: false          # true면 홈 상단 시그니처 영역에 노출 (비교글 등)
date: 2026-07-14
updated: 2026-07-14
---
```

> `country`는 태그와 별개 필드로 관리한다. 추후 `/country/taiwan/` 같은 국가별 허브 페이지를 자동 생성해 확장하기 위한 근거 데이터다(초기 빌드에선 태그처럼만 써도 됨).

### build.js 기능 (요구사항 확정)

1. `content/**/*.md` → 템플릿 적용 → `dist/` HTML 생성
2. 카테고리 인덱스·홈 자동 생성 (최신순)
3. **sitemap.xml, robots.txt 자동 생성**
4. 관련 글 3개 자동 계산 (동일 카테고리 최신 → 부족 시 태그 매칭)
5. Article JSON-LD 자동 삽입 (front matter의 date/updated 사용)
6. 로컬 미리보기: `npx serve dist` 수준이면 충분

### 필수 페이지 (애드센스 심사용)

- **소개(about)**: 0.5장 저자 프로필을 그대로 반영 — "재무 담당으로 대만·멕시코·폴란드 3개국 부임, 준비~정착 전 과정을 직접 겪은 재무 전문가". 회사명은 익명화 가능하나 역할·국가·기간·전문 배경은 유지
- **개인정보처리방침(privacy)**: 애드센스·쿠키·구글 애널리틱스 사용 고지 포함한 표준 템플릿
- **연락처(contact)**: 이메일 주소 표기 (sgh701@naver.com 또는 사이트 전용 주소, 폼 불필요)

## 9. 기존 자산 연결

- **현대해운 대만(TAICHUNG) 이사 견적서 실물 보유** = 글 2번(국제이사 견적)의 실증 자료. 금액·업체명 마스킹 후 캡처 게재 → "진짜 견적서" 콘텐츠는 검색·체류시간에 강력

- Obsidian 폴란드 정착 기록 = 글 8번 및 폴란드 관련 글의 원천 자료
- 대만·멕시코 시절 기록·사진·서류 = 해당 국가 글의 1차 자료
- 프로젝트 ②(디지털 상품)·③(인스타툰)과 주제 연결 → 추후 상호 트래픽

## 부록: CLAUDE.md 초안 (제작 시 그대로 사용)

```markdown
# 주재원노트 — 정적 사이트

## 목적
해외 주재원·해외이주 준비자를 위한 한국어 정보 사이트. 검색 유입 → 애드센스 수익.

## 저자 (콘텐츠의 근거 — 항상 이 관점 유지)
- 대만·멕시코·폴란드 3개국 주재원을 실제로 겪은 재무 담당자. 재무·세무·인사·총무를 현지에서 단독 총괄한 경험 보유.
- 강점 영역: 세금·환율·송금·자금 등 "돈" 콘텐츠 = 진짜 전문. 3개국 비교 = 경쟁자 없음.
- 회사 내부정보·특정 수치는 쓰지 않는다. 일반화된 절차·기준·본인 판단만 서술.

## 톤
- 직접 겪은 사람이 후배에게 알려주는 톤. 단정하고 실용적. 과장·감탄사 금지.
- "~했다/~하면 된다" 체. 광고성 표현 금지.

## 규칙
- 글 = content/{category}/{slug}.md 파일 1개. 빌드는 node build.js
- 모든 글에 front matter 필수 (title, description, category, tags, country, featured, date, updated)
- description은 120~155자, 글마다 고유하게
- h1은 페이지당 1개, 본문은 h2/h3 구조
- 세금·제도 글 하단에 면책 고지 자동 포함
- 경험 표시: 초안 생성 시 "[여기에 본인 경험 추가]" 마커를 본문 2곳 이상 배치
- 국가 근거가 있는 글은 어느 나라 경험인지 본문에 드러낸다 (예: "대만에서는…")
- 설정 변경은 config.json 만 수정
```
